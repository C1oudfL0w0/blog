SerialKiller是⼀个Java反序列化过滤器，可以通过黑名单与白名单的方式来限制反序列化时允许通过的类

最初的黑名单：https://github.com/ikkisoft/SerialKiller/blob/998c0abc5b/config/serialkiller.conf

<?xml version="1.0" encoding="UTF-8"?>
<!-- serialkiller.conf -->
<config>
    <refresh>6000</refresh>
    <blacklist>
	<!-- ysoserial's CommonsCollections1 payload  -->
        <regexp>^org\.apache\.commons\.collections\.functors\.InvokerTransformer$</regexp>	
	<!-- ysoserial's CommonsCollections2 payload  -->
        <regexp>^org\.apache\.commons\.collections4\.functors\.InvokerTransformer$</regexp>
	<!-- ysoserial's Groovy payload  -->	
        <regexp>^org\.codehaus\.groovy\.runtime\.ConvertedClosure$</regexp>	
        <regexp>^org\.codehaus\.groovy\.runtime\.MethodClosure$</regexp>	
	<!-- ysoserial's Spring1 payload  -->
	<regexp>^org\.springframework\.beans\.factory\.ObjectFactory$</regexp>	
    </blacklist>
    <whitelist>
        <regexp>.*</regexp>
    </whitelist>
</config>

过滤了InvokerTransformer，也就切断了cc1的链子

为了绕过这个限制，于是出现了cc3

CC1+TemplatesImpl链

在字节码那篇文章中，我们知道TemplatesImpl是⼀个可以加载字节码的类，通过调用其newTransformer()方法，即可执行这段字节码的类构造器

于是我们接下来尝试把这个特性运用到反序列化漏洞中来执行任意代码

只需对之前p牛的简化demo稍作修改即可：

将demo中InvokerTransformer执行的“方法”改成TemplatesImpl::newTransformer()

即把原先的

Transformer[] transformers = new Transformer[]{
    new ConstantTransformer(Runtime.getRuntime()),
    new InvokerTransformer("exec", new Class[]{String.class}, new Object[]{"C:\\Windows\\System32\\calc.exe"}),
};

改成

Transformer[] transformers = new Transformer[]{
    new ConstantTransformer(obj),
    new InvokerTransformer("newTransformer", null, null)
};

其中 obj 是TemplatesImpl链的对象

不过到这里还没完全摆脱 InvokerTransformer，接下来是CC3的内容

CC3

看一下ysoserial上的cc3，可以发现完全没有使用到InvokerTransformer来调用任意方法

而是用到了 com.sun.org.apache.xalan.internal.xsltc.trax.TrAXFilter类

TrAXFilter类

这个类没实现 Serializable，不能直接序列化

但是这个类的构造方法中调用了(TransformerImpl) templates.newTransformer()，免去了我们使用InvokerTransformer手工调用newTransformer()方法这⼀步

所以我们只要执行这个类的构造函数，拼上 TemplatesImpl 即可实现类加载命令执行

利用InstantiateTransformer调用TrAXFilter

不过，缺少 InvokerTransformer，TrAXFilter的构造方法也是无法调用的。这里会用到⼀个新的 Transformer：org.apache.commons.collections.functors.InstantiateTransformer

InstantiateTransformer也是⼀个实现了Transformer接口的类，他的作用是调用构造方法

下面的 try-catch 语句块中，if 判断传进来的参数Class类是否为空，为空则抛出异常，否则.getConstructor获取指定参数的构造器并用.newInstance调用构造函数

所以我们的目标是利用 InstantiateTransformer 来调用到 TrAXFilter 的构造方法，再利用其构造方法里的templates.newTransformer()调用到 TemplatesImpl 里的字节码

Transformer 的调用链：

Transformer[] transformers = new Transformer[]{
        new ConstantTransformer(TrAXFilter.class),
        new InstantiateTransformer(new Class[] { Templates.class }, new Object[] { obj })
};

替换到前面的demo中，也能成功触发

从而避免了使用 InvokerTransformer
